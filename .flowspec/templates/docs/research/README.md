# Research Reports

This directory contains research and validation reports generated by `/flow:research`.

## Purpose

Research reports capture technical exploration, business validation findings, and architectural analysis before implementation.

## File Naming Convention

```
{feature-slug}-{type}.md
```

**Types:**
- `technical` - Technical research and spikes
- `business` - Business validation findings
- `competitive` - Competitive analysis

**Examples:**
- `user-authentication-technical.md`
- `payment-gateway-business.md`

## Template

See `templates/research-template.md` for the standard research report format.

## Required Sections

1. Research Objective
2. Methodology
3. Findings
4. Recommendations
5. Next Steps
6. References

## Workflow

1. Run `/flow:research` after specification or assessment
2. Research report is generated in this directory
3. Findings inform PRD refinement or architecture decisions
4. Proceed to planning phase
