# Security Reports

This directory contains security assessment reports generated by `/flow:validate`.

## Purpose

Security reports document vulnerability assessments, compliance checks, and security recommendations for features.

## File Naming Convention

```
{feature-slug}-security-report.md
```

**Examples:**
- `user-authentication-security-report.md`
- `payment-gateway-security-report.md`

## Template

See `templates/security-report-template.md` for the standard security report format.

## Required Sections

1. Security Assessment Summary
2. Threat Model
3. Vulnerabilities Found
   - Critical
   - High
   - Medium
   - Low
4. Compliance Checklist
5. Remediation Recommendations
6. Sign-off

## Workflow

1. Run `/flow:validate` after implementation
2. Security report is generated in this directory
3. Review vulnerabilities and compliance status
4. Address critical and high severity issues
5. Proceed to release
