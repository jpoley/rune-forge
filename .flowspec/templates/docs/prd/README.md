# Product Requirements Documents (PRDs)

This directory contains PRDs generated by `/flow:specify`.

## Purpose

PRDs capture comprehensive requirements including user stories, acceptance criteria, functional and non-functional requirements, and success metrics.

## File Naming Convention

```
{feature-slug}.md
```

**Examples:**
- `user-authentication.md`
- `payment-gateway.md`

## Template

See `templates/prd-template.md` for the standard PRD format.

## Required Sections

1. Executive Summary
2. Problem Statement
3. User Stories (with Acceptance Criteria)
4. Functional Requirements
5. Non-Functional Requirements
6. Success Metrics
7. Dependencies
8. Risks and Mitigations
9. Out of Scope

## Workflow

1. Run `/flow:specify` after assessment is complete
2. PRD is generated in this directory
3. Backlog tasks are created from user stories
4. Proceed to research or planning phase

## Validation

PRDs are validated by `PRDValidator` for:
- Required sections presence
- User story format
- Acceptance criteria
