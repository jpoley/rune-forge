# QA Reports

This directory contains quality assurance reports generated by `/flow:validate`.

## Purpose

QA reports document test results, coverage metrics, and quality assessments for features.

## File Naming Convention

```
{feature-slug}-qa-report.md
```

**Examples:**
- `user-authentication-qa-report.md`
- `payment-gateway-qa-report.md`

## Template

See `templates/qa-report-template.md` for the standard QA report format.

## Required Sections

1. Test Summary
2. Test Results by Category
   - Unit Tests
   - Integration Tests
   - E2E Tests
3. Coverage Metrics
4. Performance Results
5. Known Issues
6. Recommendations
7. Sign-off

## Workflow

1. Run `/flow:validate` after implementation
2. QA report is generated in this directory
3. Review test results and coverage
4. Address any failing tests or coverage gaps
5. Proceed to security validation or release
