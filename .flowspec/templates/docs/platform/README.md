# Platform Design Documents

This directory contains platform and infrastructure design documents generated by `/flow:plan`.

## Purpose

Platform documents capture infrastructure decisions, deployment architecture, and operational requirements for features.

## File Naming Convention

```
{feature-slug}-platform.md
```

**Examples:**
- `user-authentication-platform.md`
- `payment-gateway-platform.md`

## Template

See `templates/platform-template.md` for the standard platform design format.

## Required Sections

1. Infrastructure Overview
2. Deployment Architecture
3. Scalability Requirements
4. Security Considerations
5. Monitoring and Observability
6. Disaster Recovery
7. Cost Estimation

## Workflow

1. Run `/flow:plan` to generate platform designs
2. Platform docs are created in this directory
3. Review infrastructure requirements
4. Proceed to implementation or operations phase
