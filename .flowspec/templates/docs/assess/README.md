# Assessment Reports

This directory contains SDD workflow assessment reports generated by `/flow:assess`.

## Purpose

Assessment reports evaluate whether a feature should use the full Spec-Driven Development (SDD) workflow, a lightweight version, or skip SDD entirely.

## File Naming Convention

```
{feature-slug}-assessment.md
```

**Examples:**
- `user-authentication-assessment.md`
- `payment-gateway-assessment.md`

## Template

See `templates/assessment-template.md` for the standard assessment report format.

## Workflow

1. Run `/flow:assess` to evaluate a feature
2. Assessment report is generated in this directory
3. Based on recommendation, proceed with appropriate workflow level
